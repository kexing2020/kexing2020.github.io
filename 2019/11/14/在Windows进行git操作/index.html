在Windows进行git操作

2019-11-14
git
1、创建Git仓库

  • (1)切换目录cd 路径

  • (2)创建目录mkdir 目录名

  • (3)在目录中创建新的git仓库git init

    • 成功则会提示：Initialized empty Git repository in

  • (4)在项目中查看生成的.git子目录ls -a

    • 成功则会提示./ ../ .git/
2、拷贝Git仓库
  • 拷贝Git仓库到本地git clone [url]
    • 例如：git clone git@github.com:schacon/simplegit.git
    • 例如：git clone git@192.168.2.117:vhotelsunion.com.git
3、基本快照

  • (1)将文件添加到缓存区git add 文件名

    • 新项目添加所有文件git add .
    • 添加整个文件夹及内容git add 文件夹名/
    • 如若成功没有任何提示
    • fatal: pathspec '文件名' did not match any files
      • 说明在增加时，本地文件下没有这个文件
      • 直接在目录中创建文件，而不是使用touch目录创建文件，然后再使用git add命令增加即可
      • 加上后缀名READ.md则不会出现这个问题

  • (2)查看项目当前状态git status：详细输出内容

    • 加入-s参数，则会简短输出结果git status-s
    • 如若未添加成功则显示：?? 文件名
    • 如若添加成功则显示：A 文件名
    • 当编辑完文件在再执行git status -s则显示：AM 文件名
      • “AM”状态表示添加文件到缓存之后又有改动

  • (3)查看执行git statis的结果的详细信息git diff

    • 显示已写入缓存语已修改但尚未写入缓存的改动的区别
    • 尚未缓存的改动：git diff
    • 查看已缓存的改动 :git diff --cached
    • 查看已缓存的与未缓存的所有改动 :git diff HEAD
    • 显示摘要而非整个 diff:git diff --stat

  • (4)将缓存区内容添加到仓库中git commit

    • (i)Git 为每一个提交都记录名字与电子邮箱地址，所以第一步需要配置用户名和邮箱地址
      • git config --global user.name 名字
      • git config --global user.email 邮箱
    • (ii)提交并提交注释git commit -m "注释"
    • (iii)执行git status

  • (5)取消已经缓存的内容git reset HEAD 文件名

    • 执行 git reset HEAD 以取消之前 git add 添加，但不希望包含在下一提交快照中的缓存

  • (6)命令行删除文件git rm 文件名

    • 不从工作区中删除文件git rm --cached 文件名

  • (7)移动或重命名一个文件、目录、软链接git mv 文件名

  • (8)推送到远程服务器的代码仓库git push

    • 推送本地分支到远程分支git push origin

  • (9)从远程代码仓库拉取到最新版本到本地仓库git pull

    • 拉取有变化的文件

  • (10)查看提交记录git log

  • (11)分组查看简要的提交记录git shortlog

  • (12)比较当前工作区中的指定文件和本地仓库中当前版本的文件有何不同git diff filename

  • (13)比较整个工作区中的文件和当前版本有何不同git diff HEAD

  • (14)把文件从暂存区域复制到工作目录，用来丢弃本地修改 git checkout

  • (15)合并分支命令git merge

4、文件操作
  • (1)在目录中创建空文件touch 文件名
  • (2)在vim编辑器中打开文件vim 文件名
    • 如若该文件不存在，则创建
  • (3)在vim编辑器中修改文件
    • 打开文件
    • 键盘输入字母i进入插入编辑模式
  • (4)在vim编辑器中保存文件
    • 按下ESC键，退出编辑模式，切换到命令模式
    • 在命令模式下键入”ZZ”或者”:wq”保存修改并且退出vim
    • 如果只想保存文件，则键入”:w"，回车后底行会提示写入操作结果，并保持停留在命令模式
  • (5)在vim编辑器中放弃所有文件修改
    • 按下ESC键进入命令模式，键入”:q!"回车后放弃修改并退出vim
    • .放弃所有文件修改，但不退出 vi，即回退到文件打开后最后一次保存操作的状态，继续进行文件操作：按下ESC键进入命令模式，键入”:e!"，回车后回到命令模式
  • (6) 查看文件内容 cat 文件名
  • (7)删除文件夹rm -r 文件夹名
  • (8)删除文件rm 文件名
5、Git分支管理
  • (1)创建分支命令git branch 分支名
  • (2)切换分支命令git checkout 分支名
  • (3)合并分支命令git merge
  • (4)列出分支git branch
  • (5)删除分支git branch -d (branchname)
  • (6)合并分支git branch
6、Git远程仓库(Github)
  • (1)添加远程库git remote add [shortname] [url]
  • (2)配置验证信息`ssh-keygen -t rsa -C “youremail@example.com“
7、Git基本流程
  • git add — git commit —git commit -m “注释”—git status —git push origin —git log —git checkout —git pull —git log

备注

  • mkdir是make directories
  • ls是list
  • vim是常用的编辑器
  • rm是remove
  • mv是move
