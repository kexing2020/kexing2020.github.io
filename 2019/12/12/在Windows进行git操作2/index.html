2019-12-12
git

Git安装配置

1、Windows安装包下载地址：https://gitforwindows.org/

2、测试安装是否成功

  • 查看版本git --version，命令行返回git version 2.17.1

3、配置用户信息

  • 配置用户名称git config --global user.name "DaRenLi"
  • 配置用户电子邮件地址git config --global user.email 1137653177@qq.com
  • 注意git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址

4、查看配置信息

  • 查看某个环境变量的设定git config user.name
  • 查看所有配置信息 git config --list
  • 查看所有全局设置git config --list --global
  • 查看所有本地设置git config --list --local
创建版本库/仓库(repository)

1、创建一个空目录mkdir testgit

2、创建仓库git init

  • 指定目录作为Git仓库git init testgit1

3、从现有 Git 仓库中拷贝项目到本地git clone git://github.com/schacon/grit.git

  • 自定义新建的项目目录名称git clone git://github.com/schacon/grit.git testgit2
Git基本快照

Git 的工作就是创建和保存你的项目的快照及与之后的快照进行对比。

1、将文件从工作区添加在到暂存区git add

  • 添加指定文件到暂存区git add filename
  • 添加所有文件到暂存区git add .
  • 添加整个文件夹及内容git add 文件夹名/
  • 命令行返回fatal: pathspec '文件名' did not match any files，解决方法：直接在目录中创建文件，不使用touch目录创建文件，再git add

2、将文件从暂存区提交到本地仓库git commit -m "提交日志"

  • 跳过git add直接commitgit commit -a
  • 没有设置 -m 选项，Git 会尝试打开一个编辑器以填写提交信息。 如果 Git 在配置中找不到相关信息，默认会打开 vim。

3、推送到远程服务器的代码仓库git push

  • git push是git push origin master的缩写，如想推送分支，则git push origin 分支名

4、从远程代码仓库拉取到最新版本到本地仓库git pull

  • 只拉取有变化的文件

5、查看本地代码仓库当前状态git status

  • 简短输出结果git status-s,

    当编辑完文件在再执行git status -s则显示：AM 文件名。”AM”状态表示添加文件到缓存之后又有改动

  • 如若未添加成功则显示：?? 文件名;如若添加成功则显示：A 文件名;

6、查看提交记录git log

  • 分组查看简要的提交记录git shortlog

7、显示已写入缓存与已修改但尚未写入缓存的改动的区别git diff

  • git status 显示上次提交更新后的更改或者写入缓存的改动， 而 git diff 一行一行地显示这些改动具体是什么

  • 尚未缓存的改动：git diff

  • 查看已缓存的改动：git diff --cached

  • 查看已缓存的与未缓存的所有改动：git diff HEAD

    在git中HEAD代表本的代码仓库中当前（最新）版本，HEAD^代表上一个版本，HEAD^^代表上上一个版本，HEAD~100代表上100个版本

  • 显示摘要而非整个 diff：git diff --stat

8、撤销已添加到暂存区的更改（工作区不变）git reset HEAD

9、删除文件git rm 文件名

  • 同时从工作区和暂存区删除文件,即本地的文件也被删除了git rm
  • 从暂存区删除文件,但是本地文件还存在，不被版本控制git rm --cached
  • 删除之前修改过并且已经放到暂存区域git rm -f 文件名

10、命令用于移动或重命名一个文件、目录、软连接git mv

  • 重命名文件git mv 文件名 新名字

11、从本地代码仓库签出最新版本覆盖本地的工作区git checkout 文件名

  • s如果工作区中有未添加到暂存区的更改则会被直接覆盖

12、将所有未提交的修改（工作区和暂存区）保存至堆栈中，用于后续恢复当前工作目录git stash

  • 查看当前stash中的内容git stash list
  • 可以只复制.git目录，通过git stash获取所有文件
Git分支管理

1、列出分支git branch

2、创建分支git branch 分支名

3、切换分支git checkout 分支名

  • 创建新分支并立即切换到该分支下操作git checkout -b 文件名

4、合并分支``git merge`

5、删除分支git branch -d 分支名

6、合并分支git branch

文件操作

1、在目录中创建空文件touch 文件名

2、在vim编辑器中打开文件vim 文件名

  • 如若该文件不存在，则创建
  • 在vim编辑器中修改文件，键盘输入字母i，进入编辑模式
  • 在vim编辑器中保存文件按下ESC键，退出编辑模式
  • 在命令模式下键入”ZZ”或者”:wq”保存修改并且退出vim
  • 如果只想保存文件，则键入”:w"，回车后底行会提示写入操作结果，并保持停留在命令模式
  • 按下ESC键进入命令模式，键入”:q!"回车后放弃修改并退出vim
  • 放弃所有文件修改，但不退出 vi，即回退到文件打开后最后一次保存操作的状态，继续进行文件操作：按下ESC键进入命令模式，键入”:e!"，回车后回到命令模式

3、查看文件内容 cat 文件名

4、删除文件夹rm -r 文件夹名

5、删除文件rm 文件名

6、切换目录cd 路径

7、查看当前路径pwd

2、mkdir是make directories

3、ls是list

4、vim是常用的编辑器

5、rm是remove

6、mv是move
